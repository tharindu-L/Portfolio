HTB: Sequel Writeup

Exploiting an exposed MariaDB instance with no root password — a real-world misconfiguration that still happens more than you'd think.

▶ Tharindu Pathmasiri ▶ July 2026 ▶ 10 min read

Another day, another box! Today we're cracking open a database with zero password protection 😄 Sequel is a great machine for anyone just getting into database exploitation — simple setup, but a very real vulnerability. Let's get into it 🚀

Step 01

Connect & Ping the Target

After connecting to HTB via OpenVPN and spawning the machine, my target IP was 10.129.97.143. First thing I always do — confirm the target is alive.

bash 
$ ping -c 4 10.129.97.143

PING 10.129.97.143 56(84) bytes of data.
64 bytes from 10.129.97.143: icmp_seq=1 ttl=63 time=217 ms
64 bytes from 10.129.97.143: icmp_seq=2 ttl=63 time=214 ms
64 bytes from 10.129.97.143: icmp_seq=3 ttl=63 time=205 ms
64 bytes from 10.129.97.143: icmp_seq=4 ttl=63 time=222 ms
4 packets transmitted, 4 received, 0% packet loss ✓

All 4 packets returned with 0% loss. Target is up and reachable. ✅

Step 02

Nmap Scan — Find Open Ports

Next, run a full service version scan to see what's running on this machine.

bash 
$ nmap -sVC -T5 10.129.97.143
Flags explained:
-sV — detect service versions  |  -sC — run default scripts  |  -T5 — fastest scan speed
output
PORT     STATE SERVICE VERSION
3306/tcp open  mysql
| Version: 5.5.5-10.3.27-MariaDB-0+deb10u1
| Auth Plugin: mysql_native_password

Only one open port — 3306, running MariaDB. MariaDB is a community fork of MySQL, widely used across web stacks. The fact that it's exposed directly to the network with no firewall is already a bad sign.

Step 03

Login to MariaDB as Root

Port 3306 open with no firewall? I tried the obvious — root with no password.

bash 
$ mysql -u root -h 10.129.97.143 --ssl=0

Welcome to the MariaDB monitor.
Server version: 10.3.27-MariaDB-0+deb10u1
MariaDB [(none)]> ✓  We're in.

No password prompt. We're inside the database server as root. This is a critical misconfiguration — in a real environment, this means full control over all data.

What I Learned

Sequel is simple, but the vulnerability it demonstrates is real and still found in the wild. An exposed database port with no root password gives an attacker complete control. Here's what to always do in a real environment:

🔒

Bind to Localhost

Set bind-address = 127.0.0.1 in MySQL config so the port never faces the network.

🔑

Strong Root Password

Never leave the root account without a password. Use a long, random credential.

🛡️

Firewall Rules

Block port 3306 from external access using UFW or iptables. Only allow trusted IPs.
